Music Room

About the project

Music room is an app that allows muscians within a local area to easily meet and share common interests. The app is built using a the geocoder gem which logs a users IP upon logging in and then returns a list of other users within that local area (at the moment it is set to 200kms). The app also has a built in chat network so users can immediately begin discussing jam sessions or just general interests as soon as they find a match

Music room was my first time working with a backend language. It was incredible to see the site working with different users finding each other appropriately. This project really opened my eyes to the power of a solid backend and I'm very eager to sink my teeth into more.

I would also like to send a shout out to Joseph Ndgundu without his tutorial I would have never gotten the chat up and running.

Languages and Libraries used

  • JavaScript
  • jQuery
  • HTML5
  • Ruby
  • Ruby on Rails
  • GeoCoder gem
  • Faye gem

YoungHort Aus

Languages and libraries used

YoungHort Aus is a site built for a not-for-profit startup which focuses on introducing kids of all ages to the horticulture industry.

The site was built with a 4 person team. The site needed to be user friendly for kids and teens while also allowing the administrator easy access to add and remove things posts,images and content. With these things in mind the team built a backend which contains a fully functioning CMS while utilising the cloudinary gem to handle the image upload and maintainence. They also incorporated the WYSIWYG froala gem to help style the blog posts and mandrill to help with auto emailing on the contact form. Finally the Slick and Tween Max JavaScript libraries were used to implement the image slider and background animation.

Languages and Libraries used

  • JavaScript
  • jQuery
  • HTML5
  • CSS3
  • Ruby
  • Ruby on rails
  • Cloudinary gem
  • Froala gem
  • Mandrill gem
  • TweenMax Library
  • Slick Library

Band Wiki

About the project

Band Wiki was a chance for me to have a crack at the Backbone framework for Javascript. The site is built around the soundcloud API which lets users search for music while at the same time storing those searches into a local database. The user then have the oppurtunity to update the bands bio and other content in an attempt to capture the vast knowledge of fan knowledge of bands...However the real reason for building it was to build something with Backbone

I found the Backbone framework has a lot of potential but I feel like I barely scratched the surface with this app. However the whole purpose of using it was to gain a greater understanding and I believe I accomplished that. The app is far from perfect but it is a labour of love for me. I will continue to work on it, to keep nutting away at it until I have mastered Backbone...If that can be done.
